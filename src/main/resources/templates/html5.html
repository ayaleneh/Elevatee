HTML5 Quick Tutorial

Ayaleneh yilma
Last update: 2015-02-13

HTML5 new features are based on HTML, CSS, DOM, and JavaScript, and reduce the need for external plugins (like Flash).

About HTML5 syntax

HTML5 syntax is compatible with both HTML4 and XHTML.
- You can close the single tag elements (META, BR, HR, INPUT, IMG) with a slash, <tag />, like in XHTML; or you can choose to not add the slash, <tag>.
- You can use both lower case, or upper case for the name of the tags and attributes.
So, all the variants bellow are valid HTML5:

<meta charset='utf-8'>

 OR, closed with slash:

<meta charset='utf-8' />

<img src='image.jpg' alt='Some text'>
 OR:
<img src='image.jpg' alt='Some text' />
 OR, with upper case:
<IMG src='image.jpg' ALT='Some text'>

<br>
 OR:
<BR>
 OR:
<br />

- In HTML5 you can choose to not specify the type attribute for the elements <script>, and <style>, it is optional. 
<script>
 // JavaScript code
</script>

<style>
 /* CSS code */
</style>

HTML5 structure

HTML5 doctype is much simpler: 
<!doctype html>
Meta charset tag is much simpler: 
<meta charset='utf-8'>

Divs are now used for styling rather than structure; HTML5 includes several new structural items that help define parts of the document.
These are the main structural elements you'll use most often in HTML5:
  • <header> - is used to define a header, or navigation aids for some part of a Web page, be it the entire page, an <article> element, or a <section> element.
    - Things you'd usually wrap in a H1, H2, Hx, <nav> tag with main links.
  • <footer> - represents a footer for the page, for a <section>, or for an <article>; and can contain information about the author, copyright information, etc.
  • <section> - represents a section of a document or application. It is used to logically group the document structure.
    It can contains header, articles, or navigation menu, footer.
  • <nav> - is a container for navigation links on a Web page. This item should be used for menu with navigation links.
    If you have a <footer> that contains navigation links, you do not need to wrap these links in a <nav> element, since the <footer> will suffice on its own.
  • <article> - is used to define an independent item on the page (or in a <section>) that can be distributed on its own, such as a news article, blog post, or comment.
  • <aside> - can be used to define a sidebar or some other content that is considered somewhat separate to the content around it. An example of this might be advertising blocks.
  • <hgroup> - used to group a set of H1, H2, H3, ... elements when the heading has multiple levels, such as a title and a subtitle

HTML5 comes with other interesting items such as the <video> and <audio>, plus new and changed elements and attributes, but this tutorial is a quick quide that presents just the basics that will let you quickly move from HTML4 or XHTML to HTML5.
